✏️ 인라인 텍스트 편집

캔버스에서 텍스트를 직접 클릭하여 편집합니다. 헤드라인 15자, 본문 50자, 보조텍스트 30자 제한이 실시간으로 표시됩니다. Tab키로 필드 간 이동이 가능합니다.

🔀 카드 순서 변경

좌측 사이드바에서 카드를 드래그 앤 드롭으로 재정렬합니다. dnd-kit 라이브러리를 사용하며 키보드 접근성도 지원합니다.

🎨 6가지 레이아웃

Center, Top-Left, Top-Right, Bottom-Left, Bottom-Right, Split — 카드 역할에 따라 최적의 텍스트 배치를 선택합니다.

💾 자동저장

편집 완료 1초 후 자동으로 Supabase에 저장됩니다. 모든 변경 이력은 edit_logs 테이블에 기록되어 AI 에이전트 품질 개선에 활용됩니다.

Part 3

개발 아키텍처

4개 레이어로 구성된 프론트엔드 중심 아키텍처입니다.

🖥️

Presentation Layer

사용자에게 보이는 UI 컴포넌트 계층. Next.js App Router 기반 페이지와 shadcn/ui 컴포넌트로 구성됩니다.

Next.js 14 Tailwind CSS shadcn/ui
🎨

Canvas Layer

1080×1080px 카드 렌더링 엔진. Fabric.js로 텍스트·이미지·오버레이를 합성하고, 인라인 편집을 처리합니다.

Fabric.js 6 dnd-kit 8
⚙️

State Layer

Zustand 스토어가 카드 데이터, 편집 상태, 자동저장을 관리합니다. 1초 디바운스로 효율적인 저장 전략을 구현합니다.

Zustand 4
🗄️

Backend Layer

Supabase가 인증, 데이터베이스(PostgreSQL), 파일 스토리지, Row-Level Security를 모두 담당합니다.

Supabase Vercel
Part 4

데이터 흐름

1

콘텐츠 에이전트가 card_spec.json 생성

7개 에이전트 파이프라인이 주제 선정 → 리서치 → 카피 → 디자인 → 이미지 → QC를 거쳐 완성된 card_spec을 Supabase에 저장합니다.

2

운영자가 Canvas Editor에서 확인

/editor/[id] 페이지에서 카드를 시각적으로 확인합니다. 1080×1080 인스타그램 규격 그대로 프리뷰됩니다.

3

필요 시 인라인 편집

텍스트 수정, 레이아웃 변경, 컬러 조정, 카드 순서 변경 등을 캔버스에서 직접 수행합니다. 모든 변경은 edit_logs에 기록됩니다.

4

승인 → 자동 발행

승인 버튼 클릭 시 status가 "approved"로 변경되고, Publisher 에이전트가 인스타그램·스레드에 자동 발행합니다.

5

피드백 루프

edit_logs에 기록된 수정 패턴을 분석하여 콘텐츠 에이전트의 품질을 지속적으로 개선합니다.
