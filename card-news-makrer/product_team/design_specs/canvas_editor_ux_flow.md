# T-D01: 캔버스 에디터 UX 플로우 다이어그램

_작성자: 제품 디자인 에이전트 (AGENT 0-D) | 작성일: 2026-03-09_

---

## 개요

캔버스 에디터의 전체 UX 플로우는 6개의 메인 상태(State)와 다양한 서브 상태로 구성됩니다. 각 상태 전환은 사용자 인터랙션 또는 시스템 이벤트에 의해 트리거됩니다.

---

## 상태 다이어그램 (State Machine)

```
┌─────────────────────────────────────────────────────────────────┐
│                     START: 페이지 로드                            │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│ 1. LOADING (로딩 상태)                                           │
│ ─────────────────────────────────────────────────────────────── │
│ • card_spec.json을 API로부터 받기                                │
│ • Supabase에서 데이터 페칭 (30초 타임아웃)                        │
│ • 폰트(Pretendard) 프리로드 시작                                 │
│                                                                 │
│ UI 표시:                                                       │
│ - 전체 화면: 스켈레톤 로더 (헤더, 좌측 리스트, 중앙, 우측 패널)  │
│ - 캔버스 영역: 회색 플레이스홀더                                  │
│ - 로딩 애니메이션: 스핀 로테이션 (primary #7B68EE)               │
│ - 상태 텍스트: "카드뉴스 로드 중..."                             │
└─────────────────────────────────────────────────────────────────┘
        │                              │
        │ [성공]                       │ [실패]
        ▼                              ▼
    ┌─────────────┐         ┌──────────────────┐
    │ READY       │         │ ERROR_LOAD       │
    │ (준비 완료)  │         │ (로드 실패)       │
    └─────────────┘         └──────────────────┘
        │                              │
        │                              │ [재시도]
        │                              │
        │                    ┌─────────▼─────────┐
        │                    │ 1초 대기 후 LOADING│
        │                    └───────────────────┘
        │
        ▼
┌─────────────────────────────────────────────────────────────────┐
│ 2. READY (준비 완료 상태)                                        │
│ ─────────────────────────────────────────────────────────────── │
│ • card_spec.json 파싱 완료                                      │
│ • 첫 번째 카드(cards[0]) 표시                                   │
│ • 모든 UI 컴포넌트 렌더링 완료                                   │
│ • 편집 가능한 상태로 진입                                       │
│                                                                 │
│ UI 표시:                                                       │
│ - Header: 주제명 + 상태 배지(draft/review/approved) 표시        │
│ - CardList (좌측): 카드 썸네일 목록 + 첫 카드 선택 강조         │
│ - Canvas (중앙): 1080×1080 카드 렌더링                         │
│ - StylePanel (우측): 컬러, 폰트, 레이아웃, 오버레이 조절 활성화 │
│ - Footer: "준비 완료" 상태 표시                                 │
└─────────────────────────────────────────────────────────────────┘
        │
        │ [사용자 편집 시작]
        ▼
┌─────────────────────────────────────────────────────────────────┐
│ 3. EDITING (편집 중 상태)                                        │
│ ─────────────────────────────────────────────────────────────── │
│ • 사용자가 텍스트/이미지/스타일 수정                             │
│ • 캔버스 위 텍스트를 클릭하여 인라인 편집 진행 중               │
│ • 스타일 패널에서 속성 변경 진행 중                             │
│ • 자동저장 타이머 시작 (디바운스 1초)                           │
│                                                                 │
│ 서브 상태:                                                     │
│ ┌────────────────────────────────────────────────────────────┐ │
│ │ EDITING_TEXT (텍스트 편집)                                  │ │
│ │ • 캔버스 위 IText 선택                                      │ │
│ │ • 더블클릭 또는 Enter → 편집 모드 시작                      │ │
│ │ • 글자 수 제한 표시 (headline 15자, body 50자)             │ │
│ │ • 자동저장 예약됨                                           │ │
│ └────────────────────────────────────────────────────────────┘ │
│ ┌────────────────────────────────────────────────────────────┐ │
│ │ EDITING_STYLE (스타일 편집)                                 │ │
│ │ • 우측 StylePanel에서 속성 조정                             │ │
│ │ • 실시간 캔버스 업데이트                                    │ │
│ │ • 컬러 팔레트, 폰트 크기, 오버레이 투명도 변경              │ │
│ └────────────────────────────────────────────────────────────┘ │
│ ┌────────────────────────────────────────────────────────────┐ │
│ │ EDITING_REORDER (카드 순서 변경)                            │ │
│ │ • CardList에서 드래그 시작                                  │ │
│ │ • dnd-kit으로 순서 변경 중 (드래그 피드백)                  │ │
│ │ • 드래그 드롭 시 배열 재정렬                                │ │
│ └────────────────────────────────────────────────────────────┘ │
│                                                                 │
│ UI 표시:                                                       │
│ - Canvas: 선택된 텍스트 또는 요소 하이라이트                    │
│ - 캐릭터 카운터: 실시간 글자 수 표시 (예: "12 / 15")          │
│ - Footer: "편집 중..." + 자동저장 상태 표시                    │
│ - 변경 아이콘: 저장 대기 상태 표시 (점멸)                      │
└─────────────────────────────────────────────────────────────────┘
        │
        │ [편집 완료, 1초 대기]
        │ → 자동저장 실행
        ▼
┌─────────────────────────────────────────────────────────────────┐
│ 4. SAVING (저장 중 상태)                                         │
│ ─────────────────────────────────────────────────────────────── │
│ • card_spec 업데이트를 Supabase에 전송                          │
│ • 편집 이력(edit_logs) 기록                                    │
│ • 요청 타임아웃: 10초                                           │
│                                                                 │
│ UI 표시:                                                       │
│ - Footer: "저장 중..." + 로딩 스피너                            │
│ - 모든 편집 컨트롤: 비활성화 (opacity 60%, pointer-events none) │
│ - HeaderState: "draft" 배지 유지                               │
└─────────────────────────────────────────────────────────────────┘
        │
        │ [성공]        [실패]
        ▼               ▼
    ┌─────────────┐  ┌──────────────────┐
    │ SAVED       │  │ ERROR_SAVE       │
    │ (저장 완료)  │  │ (저장 실패)       │
    └─────────────┘  └──────────────────┘
        │                   │
        │ [1초 후]          │ [재시도]
        │ or [재편집 시작]   │
        │                   └─► SAVING
        │
        ▼
    ┌─────────────┐
    │ READY       │ ◄─── [편집 완료, 캔버스 클릭 아웃 또는 다른 카드 선택]
    │ (준비 완료)  │
    └─────────────┘
        │
        │ [사용자 승인 버튼 클릭]
        ▼
┌─────────────────────────────────────────────────────────────────┐
│ 5. AWAITING_APPROVAL (승인 대기 상태)                           │
│ ─────────────────────────────────────────────────────────────── │
│ • 사용자가 [승인 & 발행] 버튼을 클릭                            │
│ • 최종 검토 모달 표시 (선택 사항: 변경 요약)                    │
│ • 발행 확인 요청                                                │
│                                                                 │
│ UI 표시:                                                       │
│ - 모달 오버레이 (opacity 50%, 배경 #2D2D2D)                    │
│ - 모달 제목: "카드뉴스를 발행하시겠습니까?"                     │
│ - 모달 본문: 카드 개수, 수정 횟수, 마지막 저장 시간 표시        │
│ - 버튼: [취소] [발행]                                          │
│ - 스타일: border-radius 16px, padding 32px                     │
└─────────────────────────────────────────────────────────────────┘
        │
        │ [발행 확인] or [취소]
        ▼
    ┌───────────────────┐  ┌──────────────────┐
    │ SUBMITTING        │  │ READY            │
    │ (발행 처리 중)    │  │ (준비 완료)      │
    └───────────────────┘  └──────────────────┘
        │
        ▼
┌─────────────────────────────────────────────────────────────────┐
│ 6. APPROVED (승인 완료 상태)                                     │
│ ─────────────────────────────────────────────────────────────── │
│ • card_spec status 변경: "draft" → "approved"                   │
│ • 발행 API 호출 시작 (AGENT 8~10 연동)                          │
│ • 편집 불가능하게 상태 전환                                     │
│                                                                 │
│ UI 표시:                                                       │
│ - Header 상태 배지: "approved" (초록색 #52C41A)               │
│ - 모든 편집 컨트롤: 비활성화                                   │
│ - 캔버스: 읽기 전용 (회색 처리)                                │
│ - Footer: "발행 완료. SNS 예약 게시 중..." 메시지              │
│ - 성공 토스트: "카드뉴스가 승인되었습니다!" (3초 표시)         │
└─────────────────────────────────────────────────────────────────┘
        │
        └─► [닫기 또는 목록으로 돌아가기]

┌─────────────────────────────────────────────────────────────────┐
│ ERROR_SAVE (저장 실패 상태)                                      │
│ ─────────────────────────────────────────────────────────────── │
│ 에러 원인:                                                      │
│ • 네트워크 끊김                                                 │
│ • Supabase 연결 실패                                            │
│ • 타임아웃 (>10초)                                              │
│ • 유효성 검사 실패                                              │
│                                                                 │
│ UI 표시:                                                       │
│ - 에러 토스트: 빨강 (#FF4D4F), 하단 중앙                       │
│ - 메시지: "저장 실패. 인터넷 연결을 확인하세요."               │
│ - [재시도] 버튼                                                │
│ - Footer: "저장 실패" 상태 표시                                │
│ - 편집 컨트롤: 다시 활성화                                     │
│                                                                 │
│ [재시도] → SAVING                                              │
│ [수동 다시 편집] → EDITING                                     │
└─────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────┐
│ ERROR_LOAD (로드 실패 상태)                                      │
│ ─────────────────────────────────────────────────────────────── │
│ 에러 원인:                                                      │
│ • API 404 (card_spec 존재하지 않음)                             │
│ • Supabase 연결 실패                                            │
│ • Invalid JSON                                                 │
│                                                                 │
│ UI 표시:                                                       │
│ - 전체 화면: 에러 메시지                                        │
│ - 에러 아이콘: 원형, 중앙 (24px, #FF4D4F)                     │
│ - 제목: "카드뉴스를 불러올 수 없습니다"                        │
│ - 설명: 에러 상세 정보 (기술자 용)                             │
│ - 버튼: [재시도] [목록으로 돌아가기]                           │
│ - 배경: surface #F8F7FF                                        │
│                                                                 │
│ [재시도] → LOADING (최대 3회)                                   │
│ [목록으로 돌아가기] → 카드 리스트 페이지                        │
└─────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────┐
│ REJECTED (반려 상태)                                             │
│ ─────────────────────────────────────────────────────────────── │
│ • [반려] 버튼 클릭 시 반려 사유 입력 모달                       │
│ • card_spec status: "draft"로 변경                              │
│ • edit_logs에 "REJECTED" 이벤트 기록                           │
│                                                                 │
│ UI 표시:                                                       │
│ - 모달: "반려 사유를 입력하세요" 텍스트 에어리어              │
│ - 최대 200자 제한                                              │
│ - 버튼: [취소] [반려]                                          │
│ - 반려 후 상태: READY로 복귀, 모든 편집 가능                   │
│ - 토스트: "카드뉴스가 반려되었습니다" (노란색 #FAAD14)        │
└─────────────────────────────────────────────────────────────────┘
```

## 상태 전환 트리거 테이블

| 현재 상태 | 트리거 | 다음 상태 | 액션 |
|-----------|--------|----------|------|
| START | 페이지 로드 | LOADING | card_spec API 호출 |
| LOADING | 성공 | READY | 첫 카드 렌더링 |
| LOADING | 실패 | ERROR_LOAD | 에러 토스트 표시 |
| ERROR_LOAD | [재시도] | LOADING | API 재호출 (최대 3회) |
| READY | 텍스트 클릭 or 스타일 변경 | EDITING | 편집 모드 진입 |
| READY | [승인 & 발행] | AWAITING_APPROVAL | 확인 모달 표시 |
| READY | [반려] | 반려 모달 | 반려 사유 입력 |
| EDITING | 1초 (자동저장 디바운스) | SAVING | Supabase 업데이트 |
| SAVING | 성공 | SAVED | 저장 완료 토스트 |
| SAVING | 실패 | ERROR_SAVE | 에러 토스트 표시 |
| SAVED | 1초 후 or 재편집 | READY | 준비 완료 상태 복귀 |
| ERROR_SAVE | [재시도] | SAVING | API 재호출 |
| AWAITING_APPROVAL | [발행] | SUBMITTING | status "approved"로 변경 |
| AWAITING_APPROVAL | [취소] | READY | 모달 닫기 |
| SUBMITTING | 성공 | APPROVED | 발행 완료 토스트 |
| SUBMITTING | 실패 | ERROR_SAVE | 에러 토스트, READY로 복귀 |
| APPROVED | N/A | 읽기 전용 | 편집 불가능 |

## 각 상태에서의 UI 반응형 레이아웃

### Desktop (1200px+)
- 좌측 CardList: 200px (고정)
- 중앙 Canvas: calc(100% - 520px) (가변)
- 우측 StylePanel: 320px (고정)

### Tablet (768px ~ 1199px)
- CardList: 160px 또는 접기 가능
- Canvas: 가변
- StylePanel: 280px 또는 접기 가능

### Mobile (< 768px)
- CardList: 하단 탭 or 모달
- Canvas: 전체 폭 (스케일 조정)
- StylePanel: 모달 형식

## 자동저장 플로우 상세

```
사용자 편집
    │
    ▼
[0ms] EDITING 상태 진입
    │
    ▼
[1000ms] 자동저장 디바운스 타이머 시작
    │
    ▼ (이 동안 추가 편집 발생 시 타이머 리셋)
[1000ms] SAVING 상태 진입
    │
    ├─► Supabase card_specs 테이블 UPDATE
    │
    ├─► Supabase edit_logs 테이블 INSERT
    │   (field_path, old_value, new_value)
    │
    ▼
[성공: 2초 후] SAVED → [1초 후] READY
[실패: 즉시] ERROR_SAVE
```

## 접근성 요구사항 (WCAG 2.1 AA)

### 포커스 관리
- Tab 순서: 헤더 → CardList → Canvas → StylePanel → Footer
- 포커스 인디케이터: 2px 테두리, #7B68EE (primary)
- Focus Visible: 모든 인터랙티브 요소

### 색상 대비
- 텍스트 vs 배경: 4.5:1 (정상 텍스트)
- 상태 배지 아이콘 vs 배경: 3:1

### 스크린 리더 지원
- 상태 메시지: aria-live="polite" (SAVING, SAVED, ERROR)
- 버튼 라벨: 명확한 action 설명
- 이미지 alt: canvas thumbnail alt text
