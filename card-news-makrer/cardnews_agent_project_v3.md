## 🛠️ 프로덕트 팀 상세

> 콘텐츠 제작 파이프라인 자체를 만들고 운영하는 내부 개발팀입니다.
> **PM이 팀을 리드하며 태스크를 생성·배분·추적합니다.**

---

### 📋 AGENT 0-PM — 프로덕트 매니저 (Product Manager)

**역할:** 프로덕트 팀의 리더. 시스템 개선 요구사항을 분석하고 태스크로 변환하여 디자이너·엔지니어에게 배분하며 진행 상황을 추적합니다.

**핵심 책임:**
- 운영자(당신)의 요청 → 명확한 요구사항으로 정의
- PRD(제품 요구사항 문서) 작성
- 태스크 분해 → 디자이너/엔지니어 배분
- 스프린트 계획 & 진행 상황 관리
- 완료된 기능 인수 테스트(Acceptance Test) 기준 정의
- 시스템 로그 분석 → 자율적 개선 제안

**PM의 태스크 관리 흐름:**

```
[운영자 요청 or 자율 감지]
         │
         ▼
  ┌──────────────────┐
  │  요구사항 분석    │  "캔버스 에디터에서 배경 교체가 불편하다"
  │  & PRD 작성      │  → 문제 정의, 범위 설정, 우선순위 결정
  └──────┬───────────┘
         │
         ▼
  ┌──────────────────┐
  │   태스크 분해     │  Epic → Story → Task 단위로 분해
  │   & 배분          │
  └──────┬───────────┘
         │
         ├──► 디자이너 태스크: "배경 교체 UX 플로우 설계"
         │
         └──► 엔지니어 태스크: "이미지 업로드 API 구현"
         │
         ▼
  ┌──────────────────┐
  │  진행 추적 &     │  태스크 보드 업데이트, 블로커 감지
  │  조율            │
  └──────┬───────────┘
         │
         ▼
  ┌──────────────────┐
  │  완료 검증 &     │  인수 기준 충족 확인 후 운영자에게 보고
  │  운영자 보고     │
  └──────────────────┘
```

**태스크 보드 구조 (PM이 자동 관리):**

```markdown
## 🗂️ 태스크 보드  [자동 업데이트]

### 🔴 Backlog
- [ ] [FEAT] 카드 순서 드래그앤드롭 개선
- [ ] [FEAT] 모바일 승인 화면 최적화
- [ ] [BUG] QA 에이전트 팩트체크 누락 케이스 수정

### 🟡 In Progress
- [ ] [FEAT] 배경 이미지 교체 기능 — 디자이너 (D-2)
- [ ] [FEAT] 배경 교체 API 엔드포인트 — 엔지니어 (D-3)

### 🟢 Done
- [x] [FEAT] 캔버스 에디터 MVP 구현
- [x] [FEAT] card_spec.json 스키마 정의
- [x] [FEAT] 텍스트 인라인 편집 기능

### 🔵 Blocked
- [ ] [FEAT] 스레드 예약 발행 — 스레드 API 정책 변경 대기 중
```

**PRD 문서 템플릿 (PM이 작성):**

```markdown
## PRD: {기능명}

### 배경 & 문제 정의
{현재 어떤 문제가 있는가}

### 목표
{이 기능으로 달성하려는 것}

### 범위 (In Scope / Out of Scope)

### 사용자 스토리
- AS A 운영자, I WANT TO {행동}, SO THAT {이유}

### 인수 조건 (Acceptance Criteria)
- [ ] {조건 1}
- [ ] {조건 2}

### 디자이너 요청 태스크
- {디자이너가 해야 할 것}

### 엔지니어 요청 태스크
- {엔지니어가 해야 할 것}

### 우선순위: P0 / P1 / P2
### 예상 완료: {날짜}
```

**Claude API System Prompt 핵심:**
```
당신은 AI 에이전트 시스템을 위한 프로덕트 매니저입니다.
운영자의 요청 또는 시스템 로그에서 문제를 발견하면
PRD를 작성하고 태스크를 생성하여 디자이너와 엔지니어에게 배분합니다.
모든 태스크는 명확한 인수 조건을 포함해야 하며,
진행 상황을 추적하고 블로커를 감지하면 즉시 보고하세요.
태스크 보드는 항상 최신 상태로 유지하세요.
```

**Input:** 운영자 요청 / 시스템 로그 / 파이프라인 성과 지표  
**Output:** PRD 문서 + 태스크 보드 + 진행 보고서

### 🎨 AGENT 0-D — 프로덕트 디자이너 (Product Designer)

**역할:** PM으로부터 디자인 태스크를 수령하고, UX 설계부터 UI 시안까지 산출하여 엔지니어에게 전달합니다.

**핵심 책임:**
- 사용자 경험(UX) 흐름 설계
- 와이어프레임 & UI 시안 작성 (코드 또는 명세서 형태)
- 디자인 시스템 & 컴포넌트 정의
- 캔버스 에디터 UI/UX 설계 및 개선
- 인터랙션 명세 작성 (엔지니어 핸드오프용)

**담당 업무 예시:**

```
PM 태스크: "배경 이미지 교체 UX 플로우 설계"
         │
         ▼
디자이너 산출물:
  1. UX 플로우 다이어그램
     현재: [교체 버튼] → 파일 탐색기 → 전체 재렌더링
     개선: [교체 버튼] → 드래그앤드롭 영역 + URL 입력 탭
           → 즉시 미리보기 → [적용] / [취소]

  2. UI 컴포넌트 명세
     - ImageReplacementPanel.jsx
       * Props: cardId, currentSrc, onReplace
       * 상태: idle | dragging | previewing | loading
       * 드롭존 크기: 280×280px
       * 지원 포맷: PNG, JPG, WEBP

  3. 인터랙션 명세 (엔지니어 핸드오프)
     - 드래그 진입: 드롭존 border #7B68EE + 배경 opacity 0.1
     - 드롭 완료: 즉시 미리보기 교체 (서버 저장 전)
     - [적용] 클릭: POST /api/cards/{id}/background
     - 실패 시: 토스트 에러 메시지 표시
```

**디자인 시스템 관리:**

```yaml
Design System:
  Colors:
    primary: "#7B68EE"
    surface: "#F8F7FF"
    text_primary: "#2D2D2D"
    success: "#52C41A"
    warning: "#FAAD14"
    error: "#FF4D4F"

  Typography:
    heading: "Pretendard Bold / 24px / -0.5px"
    body: "Pretendard Regular / 14px / 0.2px"
    caption: "Pretendard Regular / 12px / 0.1px"

  Spacing: 4px 기준 배수 (4, 8, 12, 16, 24, 32, 48)

  Components:
    - Button (primary / secondary / ghost / danger)
    - Card (canvas preview card)
    - Panel (side editor panel)
    - Toast (notification)
    - Modal (approval dialog)
```

**Claude API System Prompt 핵심:**
```
당신은 AI 에이전트 파이프라인의 내부 툴을 설계하는 프로덕트 디자이너입니다.
PM으로부터 태스크를 받아 UX 플로우, UI 컴포넌트 명세, 인터랙션 가이드를 작성합니다.
산출물은 엔지니어가 바로 구현할 수 있도록 구체적이고 명확해야 합니다.
운영자의 최소 개입으로 최대의 편의를 제공하는 UX를 설계하세요.
```

**Input:** PM의 디자인 태스크 + PRD  
**Output:** UX 플로우 + UI 컴포넌트 명세 + 인터랙션 가이드

### 💻 AGENT 0-E — 프로덕트 엔지니어 (Product Engineer)

**역할:** PM으로부터 개발 태스크를, 디자이너로부터 설계 명세를 받아 실제 코드를 구현하고 배포합니다.

**핵심 책임:**
- 디자이너 명세 기반 프론트엔드 구현 (React)
- API 엔드포인트 개발 (FastAPI)
- 에이전트 프롬프트 코드 작성 & 최적화
- 시스템 성능 모니터링 & 버그 수정
- 기술 부채 관리

**담당 업무 예시:**

```
PM 태스크: "배경 이미지 교체 API 구현"
디자이너 명세: ImageReplacementPanel 컴포넌트 스펙
         │
         ▼
엔지니어 산출물:

  1. 프론트엔드 컴포넌트
     /canvas_editor/frontend/components/ImageReplacementPanel.jsx

  2. 백엔드 API
     POST /api/cards/{card_id}/background
     → 이미지 검증 → 에셋 저장 → card_spec.json 업데이트
     → WebSocket으로 실시간 캔버스 반영

  3. 완료 보고 → PM에게 PR 링크 + 테스트 결과 전달
```

**기술 스택 담당 영역:**

```
Frontend:  React + Fabric.js + TailwindCSS
Backend:   FastAPI + Pillow + SQLite
AI 연동:   Claude API (claude-sonnet-4-20250514)
이미지:    이미지 생성 API (DALL-E / Flux)
SNS API:   Instagram Graph API, Threads API
스케줄러:  APScheduler
알림:      이메일 / Slack Webhook
```

**Claude API System Prompt 핵심:**
```
당신은 AI 에이전트 파이프라인을 구현하는 풀스택 엔지니어입니다.
PM의 태스크와 디자이너의 명세를 받아 실제 동작하는 코드를 작성합니다.
Claude API를 활용한 에이전트 구현, React 캔버스 에디터, FastAPI 백엔드를 담당합니다.
코드는 항상 명확한 주석과 함께 작성하고, 완료 후 PM에게 테스트 결과를 보고하세요.
```

**Input:** PM의 개발 태스크 + 디자이너의 UI 명세  
**Output:** 구현 코드 + 테스트 결과 + 완료 보고

**담당 업무:**
- 주제별 컬러 팔레트 선정
- 카드별 레이아웃 & 구도 설계
- `card_spec.json` 스타일 필드 완성
- 이미지 생성 AI용 프롬프트 작성

> ⚠️ 이 에이전트는 **카드뉴스 콘텐츠**의 디자인 방향을 담당합니다.
> 캔버스 에디터 자체의 UI 설계는 **프로덕트 디자이너(0-D)** 담당입니다.

**Input:** 카피라이터의 `card_spec.json`  
**Output:** `card_spec.json` (스타일 완성본) + 이미지 생성 프롬프트

---

### 🖼️ AGENT 5 — 이미지 제너레이터 (Image Generator)

**역할:** 배경 이미지 생성 + `card_spec.json` 완성

**담당 업무:**
- 배경 이미지 생성 (이미지 생성 API)
- `card_spec.json` `background.src` 연결
- 캔버스 에디터에 스펙 로드
- 초기 PNG 미리보기 렌더링

**Input:** 완성된 스타일 스펙 + 이미지 프롬프트  
**Output:** 배경 이미지 에셋 + `card_spec.json` 최종본

---

### 🔎 AGENT 6 — QA 에디터 (Quality Assurance Editor)

**역할:** 발행 전 품질, 정확성, 안전성 최종 검토

**담당 업무:**
- 팩트체크 (통계·수치 출처 확인)
- 정신건강 콘텐츠 안전 기준 준수
- 맞춤법/문법 교정
- `card_spec.json` 텍스트 오류 직접 수정

**검수 체크리스트:**
```
□ 출처 명시 여부
□ 정신건강 위기 표현 필터링 (자해/자살 관련)
□ 비전문가적 진단 표현 제거
□ 브랜드 톤 일치
□ 텍스트 가독성 (대비, 폰트 크기)
□ 해시태그 적절성
□ CTA 포함
```

**Input:** `card_spec.json` + 초기 PNG  
**Output:** 수정된 `card_spec.json` + 검수 보고서

---

### 📤 AGENT 7 — 퍼블리셔 (Publisher)

**역할:** 승인된 카드뉴스 SNS 자동 업로드

**담당 업무:**
- 인스타그램 Carousel 업로드
- 스레드 게시
- 업로드 시간 최적화
- 게시 완료 보고서 기록

**Input:** ⭐ 인간 승인 후 PNG + 캡션  
**Output:** 게시 완료 보고서 (URL, 시간, 플랫폼별 상태)

