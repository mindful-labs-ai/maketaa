# Canvas Editor v2.2 설계서

> 작성일: 2026-03-17
> 버전: v2.2 "AI Creation Flow"

---

## 1. 버전 목표

### 핵심 목표 2가지

1. **AI 카드뉴스 생성 플로우** — 홈에서 '새 카드 만들기' 클릭 시 step-by-step 대화형 위저드
2. **디자인 퀄리티 대폭 향상** — LongBlack 스타일 레퍼런스 기반 템플릿 시스템 도입

### 현재 문제점 (v2.1)

- '새 카드 만들기' 클릭 시 빈 카드 5장이 바로 생성 → 사용자가 뭘 해야 할지 모름
- 카드 디자인이 투박하고 매력적이지 않음 (기본 파스텔톤, 중앙정렬 위주)
- AI가 콘텐츠를 생성하는 플로우가 없음

---

## 2. AI 생성 플로우 설계

### 2.1 전체 흐름

```
홈 → [새 카드 만들기] → Step 1: 주제 선정 → Step 2: 목적/CTA → Step 3: 디자인 선택 → 생성 중... → 에디터
```

### 2.2 Step 1: 주제 선정

**화면 구성:**
- AI가 정신건강 관련 주제 5개를 카드 형태로 추천
- 각 추천 주제는 제목 + 한줄 설명으로 구성
- 하단 액션: `직접 입력하기` | `다시 추천받기`

**데이터 구조:**
```typescript
interface TopicSuggestion {
  title: string;        // "불면증과 수면 위생"
  description: string;  // "잠이 안 올 때 시도할 수 있는 과학적 수면 습관"
  tags: string[];       // ["수면", "생활습관", "스트레스"]
}
```

**AI 프롬프트 전략:**
- 계절/시기 반영 (예: 봄철 무기력, 시험기간 스트레스)
- 트렌드 키워드 반영 (번아웃, 디지털 디톡스 등)
- 이전 생성 주제와 중복 방지

**사용자 액션:**
| 액션 | 동작 |
|------|------|
| 추천 주제 클릭 | 해당 주제 선택 → Step 2로 이동 |
| 직접 입력하기 | 텍스트 입력 필드 노출, 입력 후 Step 2 |
| 다시 추천받기 | AI가 새로운 5개 주제 생성 |

### 2.3 Step 2: 콘텐츠 목적 설정

**화면 구성:**
- 선택한 주제가 상단에 표시
- "이 카드뉴스의 목적이 있나요?" 질문
- 옵션 카드 3개:
  - `정보 전달` — 지식/팁 공유 중심
  - `행동 유도` — 특정 서비스/프로그램/링크로 유도 (CTA 입력 필드 노출)
  - `알아서 만들어 주세요` — AI가 주제에 맞게 자동 결정

**CTA 입력 (행동 유도 선택 시):**
```typescript
interface CtaConfig {
  type: 'link' | 'follow' | 'comment' | 'custom';
  text: string;           // "프로필 링크에서 확인하세요"
  url?: string;           // 선택적 URL
  description?: string;   // 추가 맥락 (AI가 콘텐츠에 반영)
}
```

### 2.4 Step 3: 디자인 컨셉(템플릿) 선택

**화면 구성:**
- 템플릿 썸네일 카드 그리드 (처음에는 1개, 점진적 확대)
- 각 템플릿에 미리보기 + 이름 + 간단 설명
- 선택 시 하이라이트 + `카드뉴스 만들기` 버튼 활성화

**초기 템플릿: "LongBlack Editorial"**
- 레퍼런스 이미지 01~08 기반
- 다크 배경 + 블루 액센트 + 좌측 정렬 타이포그래피

**템플릿 데이터 구조:**
```typescript
interface DesignTemplate {
  id: string;                    // "longblack-editorial"
  name: string;                  // "에디토리얼 다크"
  description: string;           // "강렬한 다크 배경에 깔끔한 타이포그래피"
  thumbnail: string;             // 미리보기 이미지 경로
  config: TemplateConfig;        // 실제 디자인 설정
}

interface TemplateConfig {
  background: {
    type: 'solid';
    color: string;               // "#2A2A2E"
  };
  accent_color: string;          // "#4A7AFF" (블루)
  text_color: {
    primary: string;             // "#FFFFFF"
    secondary: string;           // "rgba(255,255,255,0.75)"
    accent: string;              // "#4A7AFF"
  };
  typography: {
    headline_family: string;     // "Pretendard"
    body_family: string;         // "Pretendard"
    headline_weight: number;     // 800
    body_weight: number;         // 400
  };
  layout: {
    text_align: 'left';         // 좌측 정렬 기본
    padding: number;             // 0.08 (캔버스 비율)
    marker: {                    // 블루 닷 마커
      type: 'dot';
      color: string;
      size: number;
      position: { top: number; left: number };
    };
    brand_logo?: {               // 하단 브랜드 로고 영역
      text: string;
      position: 'bottom-center';
      font_family: string;
      font_size: number;
      color: string;
    };
  };
  card_roles: Record<CardRole, RoleLayoutOverride>;
}
```

### 2.5 생성 프로세스

```
Step 3 완료 → 로딩 화면 (생성 중...)
  1. AI가 선택된 주제 + 목적 + 템플릿 기반으로 CardSpec JSON 생성
  2. 카드 5~8장 구성 (cover, 본문 3~5장, closing/cta)
  3. Supabase에 저장
  4. 에디터 페이지로 리다이렉트
```

### 4.2 새 타입 정의

```typescript
// === 생성 플로우 ===

type CreateStep = 'topic' | 'purpose' | 'design' | 'generating';

interface CreateFlowState {
  currentStep: CreateStep;
  topic: TopicSelection | null;
  purpose: PurposeConfig | null;
  designTemplate: string | null;  // template ID
}

interface TopicSelection {
  source: 'ai-suggested' | 'user-input';
  title: string;
  description?: string;
  tags?: string[];
}

type ContentPurpose = 'informational' | 'action-driven' | 'auto';

interface PurposeConfig {
  type: ContentPurpose;
  cta?: CtaConfig;
}

interface CtaConfig {
  type: 'link' | 'follow' | 'comment' | 'custom';
  text: string;
  url?: string;
  description?: string;
}

// === 디자인 템플릿 ===

interface DesignTemplate {
  id: string;
  name: string;
  description: string;
  thumbnail: string;
  config: TemplateConfig;
}

interface TemplateConfig {
  background: { type: 'solid'; color: string } | { type: 'gradient'; colors: string[] };
  accent_color: string;
  text_colors: { primary: string; secondary: string; accent: string; muted: string };
  typography: { headline_family: string; body_family: string };
  layout_defaults: {
    text_align: 'left' | 'center';
    padding: { x: number; y: number };
    marker?: { type: 'dot' | 'line'; color: string; size: number; position: { top: number; left: number } };
    brand_area?: { text: string; position: 'bottom-center'; font_size: number; color: string };
  };
}
```

### 4.3 AI 생성 API 설계

```typescript
// lib/ai-generate.ts

interface GenerateCardNewsRequest {
  topic: TopicSelection;
  purpose: PurposeConfig;
  template_id: string;
  card_count?: number;          // 기본 6장
  canvas_ratio?: CanvasRatio;   // 기본 1:1
}

interface GenerateCardNewsResponse {
  spec: CardSpec;               // 완성된 카드 스펙
  generation_meta: {
    model: string;
    prompt_tokens: number;
    completion_tokens: number;
  };
}

// API Route: POST /api/generate-card-news
```

### 4.4 홈 페이지 변경사항

현재 `handleCreateNew`를 수정하여 `/create` 라우트로 이동:

```typescript
// Before (v2.1)
const handleCreateNew = async () => {
  // 바로 빈 카드 5장 생성 후 에디터로 이동
  ...
  router.push(`/editor/${id}`);
};

// After (v2.2)
const handleCreateNew = () => {
  router.push('/create');  // AI 생성 플로우로 이동
};
```

### 4.5 CardCanvasClient 변경사항

기존 `card-templates.ts`의 role 기반 템플릿 시스템을 확장하여 `DesignTemplate`과 연동:

- `getTemplateForRole(role, templateId)` — 2번째 인자로 디자인 템플릿 ID 전달
- 블루 닷 마커, 브랜드 로고 등 고정 요소 렌더링 추가
- 키워드 하이라이트 렌더링 (텍스트 내 `**키워드**` 파싱)

### 핵심 디자인 원칙

1. **여백이 곧 디자인** — 텍스트 주변에 충분한 공간, 빈 영역을 두려워하지 말 것
2. **타이포그래피 위계** — 헤드라인 vs 본문 vs 보조텍스트 크기 차이를 확실히
3. **액센트는 절제** — 블루 하이라이트는 카드당 1~2개 키워드에만 적용
4. **좌측 정렬 기본** — 중앙 정렬은 Cover와 CTA에서만 사용
5. **일관된 그리드** — 좌측 패딩, 상단 마커 위치가 모든 카드에서 동일

### 생성 플로우 UI 디자인 원칙

1. 다크 배경 기반 (카드뉴스 결과물과 톤 통일)
2. 선택지는 카드 형태로 제공 (hover 시 미세한 lift 효과)
3. Step 간 부드러운 전환 (fade + slide)
4. 프로그레스 인디케이터는 미니멀하게 (도트 또는 얇은 라인)
5. 모바일에서도 사용 가능한 터치 친화적 크기
