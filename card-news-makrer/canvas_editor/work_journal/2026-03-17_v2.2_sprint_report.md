**BUG-001 상세 분석**

**증상:** Step 1→2 전환 시 콘텐츠가 DOM에는 있지만 opacity: 0 상태로 렌더링되어 시각적으로 보이지 않음.

**원인:** `CreateWizard.tsx`의 `StepContent` 컴포넌트에서:
```
setDisplayStep(step) 호출
  ↓ React re-render
  ↓ cleanup이 innerTimerRef 클리어
  ↓ setOpacity(1) 타이머 취소
  ↓ opacity는 0에 머무름
```

**해결책:** fade-in을 별도 useEffect로 분리, requestAnimationFrame 사용:
```typescript
// useEffect 1: fade out + swap content
React.useEffect(() => {
  if (step === displayStep) return;
  setOpacity(0);
  const timer = setTimeout(() => {
    setDisplayStep(step);
  }, 200);
  return () => clearTimeout(timer);
}, [step, displayStep]);

// useEffect 2: fade in (독립적)
React.useEffect(() => {
  if (displayStep === step && opacity === 0) {
    const raf = requestAnimationFrame(() => setOpacity(1));
    return () => cancelAnimationFrame(raf);
  }
}, [displayStep, step, opacity]);
```

## 기술 설계 실현

### 1. 상태 관리 흐름

```
Home: "새 카드 만들기" 클릭
  ↓
/create → useCreateStore 초기화
  ↓
CreateWizard 렌더링
  ├─ Step 1 (topic): suggestTopics() API 호출 → TopicSuggestion[] 로드
  ├─ Step 2 (purpose): 목적 선택 + CTA 입력
  ├─ Step 3 (design): 템플릿 선택 (현재 1개)
  ↓
Step 3 완료: generateCardNews() API 호출
  ↓
GeneratingScreen: 4단계 프로그레스 메시지 표시
  ↓
CardSpec 생성 + Supabase 저장
  ↓
/editor/{id} 리다이렉트
  ↓
CardCanvasClient: LongBlack 템플릿으로 6장 렌더링
```

### 2. 타입 안전성

**타입 계층:**
```
types/index.ts (중앙)
  ├── CreateStep, TopicSuggestion, TopicSelection
  ├── PurposeConfig, CtaConfig
  ├── DesignTemplate, TemplateConfig, RoleLayoutOverride
  ├── GenerateCardNewsRequest/Response
  └── CardSpec (기존)

useCreateStore (상태 타입)
  ├── useCreateStore<CreateStoreState>
  └── State의 모든 필드 타입 정의

lib/ai-generate.ts (API)
  ├── GenerateCardNewsParams (요청)
  └── 반환값 타입

lib/design-templates.ts (설정)
  ├── LONGBLACK_EDITORIAL: DesignTemplate
  └── 실행 시 타입 검증
```

### 3. API 설계

**POST /api/suggest-topics**
```typescript
Request: {}
Response: {
  topics: TopicSuggestion[]
}
```

**POST /api/generate-card-news**
```typescript
Request: {
  topic: TopicSelection
  purpose: PurposeConfig
  template_id: string
  card_count?: number
  canvas_ratio?: '1:1' | '4:5' | '9:16'
}
Response: {
  spec: CardSpec
  generation_meta: {
    model: string
    prompt_tokens: number
    completion_tokens: number
  }
}
```

### 3. 버그 수정 (심각도 HIGH↑)

#### BUG-001: Step 전환 시 opacity 버그 (CRITICAL)

**증상:** Step 1→2 이동 시 콘텐츠가 DOM에 렌더링되지만 opacity: 0 상태로 화면에 보이지 않음

**원인:** 동일 useEffect에서 fade-out과 fade-in 로직이 간섭
```
setOpacity(0) 호출
  ↓ setTimeout → setDisplayStep(step)
  ↓ React re-render
  ↓ cleanup이 타이머 취소
  ↓ setOpacity(1) 실행 안됨
  ↓ opacity: 0에서 멈춤
```

**해결:** useEffect를 2개로 분리 + requestAnimationFrame 사용

```typescript
// useEffect 1: fade out + swap content
React.useEffect(() => {
  if (step === displayStep) return;
  setOpacity(0);
  const timer = setTimeout(() => {
    setDisplayStep(step);
  }, 200);
  return () => clearTimeout(timer);
}, [step, displayStep]);

// useEffect 2: fade in (독립적)
React.useEffect(() => {
  if (displayStep === step && opacity === 0) {
    const raf = requestAnimationFrame(() => setOpacity(1));
    return () => cancelAnimationFrame(raf);
  }
}, [displayStep, step, opacity]);
```

**파일:** `components/create-flow/CreateWizard.tsx` 라인 161~180

#### BUG-002: 생성 후 에디터 리다이렉트 안됨

**원인:** mountedRef 가드가 리다이렉트 타임아웃을 차단

**해결:** 리다이렉트는 mountedRef 검증 제거 (unmount 중에도 navigation 필수)

```typescript
redirectTimerRef.current = setTimeout(() => {
```

```typescript
  if (didStartRef.current) return;  // ← 중복 호출 방지
  didStartRef.current = true;
  mountedRef.current = true;
  startGeneration();
  return () => {
    mountedRef.current = false;
    cleanup();
  };
}, []);
```

**파일:** `components/create-flow/GeneratingScreen.tsx` 라인 96~106

#### BUG-005: setTimeout 메모리 누수

**증상:** 컴포넌트 unmount 시 timerRef가 정리되지 않아 메모리 누수

**해결:** fadeTimerRef, redirectTimerRef 추가로 명시적 정리

```typescript
const fadeTimerRef = useRef<ReturnType<typeof setTimeout> | null>(null);
const redirectTimerRef = useRef<ReturnType<typeof setTimeout> | null>(null);

const cleanup = () => {
  if (timerRef.current) clearTimeout(timerRef.current);
  if (intervalRef.current) clearInterval(intervalRef.current);
  if (progressIntervalRef.current) clearInterval(progressIntervalRef.current);
  if (fadeTimerRef.current) clearTimeout(fadeTimerRef.current);     // ← 추가
  if (redirectTimerRef.current) clearTimeout(redirectTimerRef.current); // ← 추가
};
```

**파일:** `components/create-flow/GeneratingScreen.tsx` 라인 88~114
