# v2.1 버그 수정 보고서

**날짜:** 2026-03-17
**버전:** v2.0 → v2.1 패치
**테스트 환경:** 로컬 dev 서버 (`npm run dev`)

---

## 개요

v2.0 출시 후 로컬 테스트에서 발견된 **10개 버그**를 수정했습니다. 에디터의 핵심 렌더링 이슈(텍스트 표시, 캔버스 스케일링, 레이아웃)와 인증 플로우 문제를 해결했습니다.

---

## 해결한 버그들

### 1. 제목(headline) 텍스트 캔버스에서 안 보임

**원인:**
- `fontWeight: 'bold'`가 Fabric.js v6 Textbox에서 Pretendard 폰트와 함께 사용 시 렌더링 실패
- 또한 fontSize 72px가 과도하여 캔버스를 벗어남
- 웹폰트의 bold variant가 별도로 로드되지 않으면 발생 가능

**해결:**
- `fontWeight` 제거 (텍스트는 기본 Regular로 표시)
- fontSize 최대 48px 제한

**파일:** `components/CardCanvasClient.tsx`

**인사이트:**
Fabric.js Textbox에서 특정 폰트 + bold weight 조합이 텍스트를 완전히 렌더링하지 못하는 경우가 있습니다. 웹폰트의 bold variant가 별도로 로드되어야 하는데, 그렇지 않으면 fontWeight 속성을 무시합니다. 간단한 해결책은 fontWeight를 제거하는 것입니다.

---

### 2. 부제(sub_text) 특정 카드에서 안 보임 (카드 1,5에서는 보이고 2,3,4에서는 안 보임)

**원인:**

*원인 1:* 카드 생성 시 2~4번 카드에 `sub_text` 필드 누락
- cover/cta만 포함되어 다른 필드는 정의되지 않음

*원인 2:* 해결 후에도 안 보임
- 템플릿의 `lineHeight`, `charSpacing` 속성이 특정 역할 템플릿에서 Fabric.js Textbox 렌더링 실패 유발
- 배열 구조 내에서 변수 스코프 충돌로 인해 `lineHeight` 값이 다른 텍스트 필드에서 참조됨

**해결:**
- 모든 카드에 `headline`, `body`, `sub_text` 기본 포함
- sub_text 렌더링에서 `lineHeight`, `charSpacing` 제거
- 변수 스코프 충돌 방지를 위해 변수명 prefix 분리 (`subLineHeight` → 제거)

**파일:** `components/CardCanvasClient.tsx`, `app/page.tsx`

**인사이트:**
Fabric.js Textbox는 `lineHeight`나 `charSpacing` 값이 undefined일 때 내부적으로 NaN 계산을 발생시켜 렌더링이 실패할 수 있습니다. 명시적 값을 넣거나 아예 옵션에서 제거해야 안전합니다. 또한 배열 내에서 각 항목의 변수명이 명확히 분리되어야 스코프 충돌을 방지할 수 있습니다.

### 3. 텍스트 색상이 배경과 동일하여 안 보임

**원인:**
- `resolveTextColor` 함수가 `hasImageBg` 조건으로 templateFill을 무시
- cover 등 어두운 solid 배경에서 어두운 fallback 색 사용
- content 역할 템플릿의 sub_text fill이 흰색 계열이라 밝은 배경에서 안 보임

**해결:**
- 배경색 luminance 자동 감지 로직으로 교체
- 밝은 배경(`luminance > 0.5`) → `#2D2D2D` (어두운 텍스트)
- 어두운 배경(`luminance <= 0.5`) → `#FFFFFF` (밝은 텍스트)
- 사용자 팔레트 색상 최우선

**파일:** `lib/card-templates.ts`

**인사이트:**
카드 역할별 템플릿의 fill 색상은 특정 배경 유형(이미지/단색)을 전제로 설계되었기 때문에, 배경 유형이 다르면 텍스트가 안 보일 수 있습니다. 배경 밝기 기반 자동 대비가 가장 안전한 방법입니다.

### 4. 캔버스 비율 변경 시 실제 캔버스 크기 안 바뀜

**원인:**
- CSS `objectFit: contain`은 `<canvas>` 요소에서 작동하지 않음 (canvas는 replaced element가 아님)
- Fabric.js가 생성한 wrapper div도 고정 크기로 설정됨
- 캔버스가 원본 크기(1080px+)로 렌더링되어 화면에서 잘리고 스크롤 발생

**해결:**
- CSS `transform: scale()` + `ResizeObserver` 방식으로 전환
- 컨테이너 크기를 측정하여 scale factor 계산
- canvas wrapper에 `transform: scale(factor)` 적용
- `transformOrigin: 'center center'`로 중앙 기준 스케일링

**파일:** `components/CardCanvasClient.tsx`

**코드 예시:**
```typescript
const scale = containerWidth / CANVAS_WIDTH;
canvasWrapperRef.current.style.transform = `scale(${scale})`;
canvasWrapperRef.current.style.transformOrigin = 'center center';
```

**인사이트:**
Fabric.js canvas는 CSS replaced element가 아니므로 `objectFit`이 작동하지 않습니다. CSS `transform: scale()`이 유일하게 안정적인 스케일링 방식입니다. 내부 좌표계는 원본 유지되므로 텍스트/이미지 위치에 영향을 주지 않습니다.

### 5. 캔버스 우측 치우침

**원인:**
- 좌측 사이드바(224px)와 우측 사이드바(256px) 너비 불균형
- 캔버스 wrapper의 CSS가 중앙 정렬을 제대로 적용하지 못함
- flex 레이아웃에서 `justify-center`와 `transform: scale()`의 상호작용 미흡

**해결:**
- CSS transform 방식으로 전환하면서 자연스럽게 해결
- 부모 컨테이너: `flex items-center justify-center`
- canvas wrapper: `transformOrigin: 'center center'`로 정확한 중앙 배치
- 좌우 패딩 제거

**파일:** `components/CardCanvasClient.tsx`

---

### 6. 레이아웃 변경 시 텍스트 아트보드 밖으로 이탈

**원인:**

*원인 1:* `fabric.IText`가 `width` 속성을 줄바꿈 기준으로 사용하지 않아 텍스트가 무한 확장

*원인 2:* `clampTop`은 수직만 제한하고 수평 제한 없음

*원인 3:* `LAYOUT_POSITIONS`의 left/originX 조합이 부적절하여 일부 레이아웃에서 위치 계산 오류

**해결:**
- `fabric.IText` → `fabric.Textbox` 전환 (width 기준 자동 줄바꿈)
- `LAYOUT_POSITIONS`에 `originX`, `textAlign`, `widthRatio` 세분화
- 수평 경계 검증 로직 추가

**파일:** `components/CardCanvasClient.tsx`, `lib/card-templates.ts`

**인사이트:**
Fabric.js에서 자유로운 텍스트 편집이 필요하면 `Textbox`를 사용해야 합니다. `IText`는 무한 확장될 수 있고, 줄바꿈 기준이 불명확합니다. `Textbox`는 명시적인 width 속성으로 자동 줄바꿈을 제공합니다.

### 7. 폰트 크기가 9:16 비율에서 너무 작음

**원인:**
- 템플릿 폰트 크기가 1080x1080 기준으로 설계됨
- 1080x1920(9:16)에서는 높이만 확대되므로 폰트가 축소 표시됨
- 실제 화면 크기가 절반 이하로 표시됨

**해결:**
- `fontScale` 계수 도입
- `cvH > 1200 → 1.5배`
- `cvH > 1080 → 1.2배`
- 기본값: 1.0배

**파일:** `lib/card-templates.ts`

**코드 예시:**
```typescript
const fontScale = cvH > 1200 ? 1.5 : cvH > 1080 ? 1.2 : 1.0;
const scaledFontSize = fontSize * fontScale;
```

**인사이트:**
템플릿이 정사각형 기준(1:1)으로 설계되었을 때, 다른 비율에서 폰트 크기를 동적으로 조정해야 가독성이 유지됩니다. 캔버스 높이를 기준으로 선형 스케일링이 가장 직관적입니다.

---

### 8. 빈 카드에서 텍스트 추가/편집 불가

**원인:**
- 빈 텍스트 필드(`''`)가 falsy로 처리되어 Textbox가 생성되지 않음
- 클릭 대상이 없어서 편집 모드 진입 불가
- 필드 없음(`undefined`)과 비어있음(`''`)을 구분하지 않음

**해결:**
- 빈 필드에 회색 이탤릭 플레이스홀더 표시
- `undefined` = 필드 없음, `''` = 필드 있지만 비어있음 구분
- StylePanel에 "콘텐츠 편집" 섹션 추가
- 각 필드를 개별 텍스트박스로 표시

**파일:** `components/CardCanvasClient.tsx`, `components/StylePanel.tsx`

**플레이스홀더 예시:**
```typescript
const displayText = text || '텍스트를 입력하세요';
const textbox = new fabric.Textbox(displayText, {
  fill: text ? '#000000' : '#AAAAAA',
  fontStyle: text ? 'normal' : 'italic',
  opacity: text ? 1 : 0.6,
});
```

### 9. 로그인 후 대시보드 이동 불가

**원인:**
- `lib/supabase.ts`의 브라우저 클라이언트가 `@supabase/supabase-js`의 `createClient` 사용
- 이 방식은 세션을 localStorage에 저장
- `middleware.ts`는 `@supabase/ssr`로 쿠키에서 세션 읽음
- 세션 저장소 불일치로 미들웨어에서 인증 실패

**해결:**
- 브라우저 클라이언트를 `@supabase/ssr`의 `createBrowserClient`로 변경
- 쿠키 기반 세션으로 통일
- 미들웨어와 클라이언트의 세션 저장소 동일화

**파일:** `lib/supabase.ts`

**코드 예시:**
```typescript
// 변경 전 (불안정)
import { createClient } from '@supabase/supabase-js';
const supabase = createClient(url, key);

// 변경 후 (안정)
import { createBrowserClient } from '@supabase/ssr';
const supabase = createBrowserClient(url, key);
```

**인사이트:**
Supabase 인증에서 미들웨어와 클라이언트를 사용할 때 반드시 같은 세션 저장소를 사용해야 합니다. 브라우저에서는 `@supabase/ssr`의 `createBrowserClient`를 사용하여 쿠키 기반 세션으로 통일하면 미들웨어와 자동 동기화됩니다.

### 10. dev 서버 시작 시 `Cannot find module './682.js'` 에러

**원인:**
- `.next` 빌드 캐시 corruption
- 많은 파일 변경 후 webpack chunk ID가 변경되었지만 stale 참조 남음

**해결:**
- `rm -rf .next` 후 재시작
- 다음 `npm run dev` 시 캐시 재생성

**명령어:**
```bash
rm -rf canvas_editor/.next
npm run dev
```

**인사이트:**
Next.js는 webpack chunk ID를 기반으로 캐시합니다. 대규모 파일 변경 후 캐시가 일관성을 잃으면 stale 참조가 발생합니다. 빌드 캐시를 삭제하는 것이 가장 빠른 해결책입니다.

## 추가 개선: 새 카드 기본 설정 통일 (버그 #11)

**문제:** 새 카드뉴스 생성 시 1~5번 카드의 배경색, 폰트 크기가 역할별 템플릿에 따라 모두 달랐음. 사용자가 아무 설정도 하지 않은 상태에서 카드마다 다른 모습으로 보여 혼란 유발.

**원인:** 각 카드의 `role`(cover, empathy, cause 등)에 따라 `getTemplateForRole()`이 다른 템플릿을 반환하여 배경색(`solid_fallback`), 폰트 크기(`fontSize`)가 모두 달랐음.

**해결:** 카드 생성 시 모든 카드에 동일한 명시적 스타일을 설정:
- 배경색: `#F0F4F8` (밝은 블루그레이)
- 텍스트색: `#2D2D2D`
- 제목 폰트: Pretendard 36px
- 본문 폰트: Pretendard 24px
- 사용자 팔레트가 설정되면 템플릿 기본값보다 우선 적용되므로, 통일된 초기 상태에서 시작 후 개별 커스터마이징 가능

**변경 파일:** `app/page.tsx` — `handleCreateNew`에서 `color_palette`, `font` 명시적 설정
