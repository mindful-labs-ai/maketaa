# Canvas Editor v2.3 스프린트 보고서

> 날짜: 2026-03-18
> 버전: v2.3 "Card Structure Redesign & Design Enhancement"
> 상태: **완료**

---

## 스프린트 요약

v2.3은 카드뉴스의 **콘텐츠 구조 전면 개편**과 **디자인 품질 대폭 향상**을 핵심 목표로 진행했습니다.

---

## 1. 카드 역할 구조 개편

### Before (v2.2)
- 9개 고정 역할: cover, empathy, cause, insight, solution, tip, closing, source, cta
- 항상 6장 고정

### After (v2.3)
- **3개 역할**: `cover` / `content` / `end`
- content 카드는 **4~8장** (AI가 주제에 따라 결정)
- content 카드에 **2가지 레이아웃**: `basic` (서술형) / `memo` (리스트형, 세트당 최대 1장)

### 변경 파일
| 파일 | 변경 내용 |
|------|-----------|
| `types/index.ts` | `CardRole` → 3종, `ContentLayout` 타입 추가, `Card`에 `content_layout` 필드 |
| `schemas/card_spec.schema.json` | role enum 3종, `content_layout` 속성 추가 |
| `app/api/generate-card-news/route.ts` | 프롬프트 전면 재작성, Gemini 변환/폴백 갱신 |
| `lib/card-templates.ts` | 3종 템플릿 + memo 별도 템플릿 |
| `components/CardCanvasClient.tsx` | cover/content(basic)/content(memo)/end 렌더링 분기 |
| `components/CardList.tsx` | 3종 역할 배지 + 메모 카드 구분 |
| `components/SortableCardItem.tsx` | 동일 배지 업데이트 + 옛 role 폴백 처리 |

## 3. 디자인 개선

### 폰트 크기 (1080px 캔버스 기준, `renderLongBlackCard`)

| 카드 | 제목 | 본문 | 부제 |
|------|------|------|------|
| 표지 (cover) | 80px | — | 32px |
| 본문 (content basic) | 42px | 38px | 28px |
| 메모 (content memo) | 44px | — | — |
| 마무리 (end) | 42px | 34px | 28px |

### 동적 텍스트 배치
- 본문 카드(basic): headline 높이 측정 후 body 위치 동적 계산 (40px 간격)
- 마무리 카드(end): headline 높이 측정 후 body 위치 동적 계산 (32px 간격)
- 제목이 2줄 이상이 되면 body가 자동으로 아래로 밀림

### 메모 카드 레이아웃 (reference: card_news_design_07)
- 숫자: 이모지(❶❷) → **파란/초록 원 + 흰 숫자** (Circle + Text)
- 원 크기: radius 22px, 숫자 26px
- 제목과 숫자 수직 중앙 정렬 (`originY: 'center'`)
- 아이템 간격: 박스 높이를 아이템 수로 균등 분할
- 메모 박스 배경: 하드코딩 `#B8D4F0` → `accentColor + '25'` (템플릿 포인트 컬러 기반)

### 마무리 카드 레이아웃 (reference: card_news_design_08)
- 상단 45%: 이미지 영역 (이미지가 영역 내 중앙 정렬)
- 하단 55%: 다크 배경(`#1E1E24`) + 텍스트
- 블루/그린 액센트 세퍼레이터 라인

### 브랜드
- `'LongBlack'` / `'CardNews'` → `'@mindthos'` (Georgia, serif)

### 비율별 폰트 스케일
- 모든 비율(1:1, 4:5, 9:16)에서 동일한 폰트 크기 (`fs = 1`)

### 폰트 변경 반영
- `document.fonts.ready` 후 Fabric.js 텍스트 `initDimensions()` + `renderAll()` 호출
- 글꼴 변경 시 캔버스에 즉시 반영

## 4. 이미지 자동 선택

### 3단계 폴백 구조
```
Unsplash 검색 → Unsplash 범용 검색 → Gemini 이미지 생성
```

### Gemini 기반 키워드 생성
- 하드코딩 `KO_EN_QUERY_MAP` (19개 매핑) → **`generateImageKeywords()` 함수**로 교체
- Gemini가 한국어 텍스트를 분석해 영어 3-4단어 Unsplash 검색 키워드 생성
- 표지: `topic.title + tags` 기반, `dark moody` 분위기
- 마무리: 마무리 카드 `headline + body` 기반, `hopeful warm` 분위기
- 핵심 사물/상황을 반드시 포함하도록 프롬프트 설계

### Gemini 이미지 생성 (최종 폴백)
- 모델: `gemini-3-flash-preview` (responseModalities: TEXT + IMAGE)
- base64 data URL로 반환 → Fabric.js 캔버스에 직접 렌더링

## 6. AI 프롬프트 개선

### 카드 생성 프롬프트 (`generate-card-news`)
- 본문 카드 수 다양성 강화: "예시 JSON의 카드 수를 그대로 따르지 마세요" 명시
- JSON 예시 축소: content 5장 → 1장 + 주석
- 각 카드 완결성 지시: "독자가 한 카드만 봐도 핵심을 이해할 수 있어야 합니다"
- 메모 카드 없이 basic만으로 구성 가능하다고 명시
- 최소 카드 검증: 6장 → 5장 (cover + 3content + end)

### 주제 추천 프롬프트 (`suggest-topics`)
- 25개 카테고리 풀에서 매번 3개 랜덤 선택
- 랜덤 시드(`Math.random()`) 추가
- 시즌 주제 1-2개 + 자유 주제 3-4개 구성
- "~하는 법" 같은 뻔한 제목 지양, 구체적 상황/감정 기반 제목 유도

## 7. 환경 설정

### env 파일 정리

- 시스템 환경변수가 `.env.local`을 덮어쓰는 문제 발견

### 모델 통일
- 모든 Gemini 호출을 `gemini-3-flash-preview`로 통일
