# Canvas Editor MVP - QA 계획서 및 체크리스트

> 작성일: 2026-03-16
> 대상 버전: canvas-editor-mvp 0.1.0
> 기술 스택: Next.js 14.2.3, TypeScript 5.3, Tailwind CSS 3.4, Fabric.js 6, Supabase, Zustand 4.4, dnd-kit 6/8

---

## 1. QA 목표 및 범위

### 1.1 목표

- MVP 핵심 워크플로우(대시보드 조회 -> 에디터 진입 -> 카드 편집 -> 저장)의 안정성 검증
- Fabric.js 캔버스 렌더링 정확도 확인 (1080x1080 기준)
- 드래그 앤 드롭, 텍스트 편집, 스타일 변경 등 주요 인터랙션의 정상 동작 확인
- 에러 처리 경로(네트워크 실패, 잘못된 ID 등)의 graceful degradation 확인

### 1.2 범위

| 범위 내 (In-Scope)                     | 범위 외 (Out-of-Scope)             |
|-----------------------------------------|------------------------------------|
| 대시보드 (`/`) 페이지 전체 기능          | 인증/로그인 기능 (현재 비활성화)    |
| 에디터 (`/editor/[id]`) 페이지 전체 기능 | 새 카드 생성 기능 (미구현, disabled) |
| CardList 드래그 앤 드롭 재정렬           | 퍼블리시/리포트 워크플로우          |
| CardCanvasClient Fabric.js 렌더링       | 서버 사이드 API (Server Actions)    |
| StylePanel 모든 섹션                    | 멀티유저 동시 편집                  |
| TextEditModal 텍스트 편집               | 모바일 네이티브 앱                  |
| 키보드 내비게이션 (ArrowLeft/Right/Esc)  | 접근성 전면 감사 (WCAG AA)         |

### 1.3 제외 사항 근거

- **로그인**: `middleware.ts:12-14`에서 `/login` 접근 시 `/`로 리다이렉트 처리됨. 인증 게이트 비활성화 상태이므로 기능 테스트 불필요.
- **새 카드 생성**: `app/page.tsx:148`에서 버튼이 `disabled` 상태이며 `handleCreateNew`는 alert만 표시.

## 5. 알려진 리스크 및 주의사항

### 5.1 Fabric.js 타입 이슈

- `@types/fabric` v5와 `fabric` v6 사이 타입 불일치 존재 (`package.json:21,30`)
- 빌드 설정에서 `ignoreBuildErrors: true`로 우회 중 (CLAUDE.md 참조)
- **QA 영향**: 런타임 에러는 타입 체크에 잡히지 않으므로 수동 테스트 필수

### 5.2 Canvas dispose 메모리

- `CardCanvasClient.tsx:216-218`에서 `fabricCanvas.dispose()` 호출하나, 이미지 객체의 완전한 해제 여부 미확인
- **QA 영향**: 장시간 사용 시 메모리 누수 가능성 (P-04 테스트로 검증)

### 5.3 인증 비활성화

- 현재 middleware에서 모든 요청 통과 (`middleware.ts:16`)
- `/login` 페이지 코드는 존재하나 접근 불가 (리다이렉트)
- **QA 영향**: 인증 관련 테스트는 MVP 범위 외

### 5.4 자동 저장 (Auto-Save)

- Zustand store에 autoSave 관련 상태 존재 (`useAutoSaveStatus`)
- 스타일/텍스트 변경 시 Supabase에 자동 저장되는 플로우
- **QA 영향**: 저장 실패 시 사용자 알림 부재 가능성 (console.error만 존재)
